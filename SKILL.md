# 论文大纲生成

基于跨学科论文结构研究成果（5领域50篇真实论文章节树分析），按学科认知范式生成规范的研究论文大纲。

---

### 三段式骨架（跨领域通用）

所有论文的顶层结构都是三段式：

```
引言（Introduction）→ 核心章节（Core）→ 结论（Conclusion）
```

- **引言**：建立研究空间（CARS模型驱动）
- **核心章节**：领域结构原型决定组织形式
- **结论**：收束论证/总结发现/展望未来

### 四种结构原型路由

根据用户的研究领域和论文类型，路由到对应的结构原型：

#### 原型1：论证-证明型（Claim-Evidence）
**适用领域**：物理学、经济学理论、cs.LG理论论文、数学
**核心逻辑**：提出命题→提供证明→验证结论
**典型结构**：`Introduction → Model/Theory → Experiment/Evidence → Conclusion`
**关键特征**：方法论单独成章、理论/推导单独成章、实验验证为必选项

**章节模板**：
```
1 引言
2 相关理论基础（可选：如领域已有成熟框架）
3 方法/模型
  3.1 问题形式化
  3.2 模型设计
  3.3 理论分析（定理/命题）
4 实验/验证
  4.1 实验设置
  4.2 主结果
  4.3 消融/敏感性分析
5 讨论（可选）
6 结论与展望
```

#### 原型2：流程-报告型（Process-Report）
**适用领域**：临床医学、计算生物学、部分cs.CL/cs.SE、实证社会学
**核心逻辑**：如实报告研究过程——做了什么→发现了什么→意味着什么
**典型结构**：`Introduction → Methods → Results → Discussion → Conclusion`
**关键特征**：方法论独立成章、数据说明独立、讨论节独立、最接近经典IMRaD

**章节模板**：
```
1 引言
2 文献综述（可选）
3 研究方法
  3.1 数据来源
  3.2 研究设计
  3.3 分析方法
4 研究结果
  4.1 描述性结果
  4.2 核心发现（按RQ或假设组织）
  4.3 稳健性/敏感性分析
5 讨论
6 结论
```

#### 原型3：系统-评估型（System-Evaluation）
**适用领域**：计算机工程（DB/NI/CV工程类）、软件工程、系统设计论文
**核心逻辑**：展示系统设计和实际效果
**典型结构**：`Introduction → Problem → Architecture/Design → Implementation → Evaluation → Conclusion`
**关键特征**：Problem Statement独立成章、System Architecture独立成章、Related Work位置灵活（可在末尾）

**章节模板**：
```
1 引言
2 相关工作（可在末尾）
3 问题定义/形式化
4 系统设计/方法
  4.1 整体架构
  4.2 核心模块设计
  4.3 关键算法/机制
5 实验评估
  5.1 实验设置
  5.2 主结果
  5.3 消融/效率分析
6 讨论（可选）
7 结论
```

#### 原型4：叙事-论证型（Narrative-Argument）
**适用领域**：人文学科（文学、历史、哲学、艺术史）、定性研究
**核心逻辑**：按思想发展的内在逻辑组织，段落标题=论证命题
**典型结构**：`Introduction → 论证展开(主题驱动) → Conclusion`
**关键特征**：无Methodology节、标题为论题陈述而非方法描述、章节数量灵活（3-9个）、结论名称多样

**章节模板**：
```
1 引言（问题提出+立场声明+对手立场）
2 概念辨析/语境建立
3 核心论证一（正论/概念展开）
4 核心论证二（批判/反驳已有观点）
5 回应可能的反对意见（可选）
6 结论（收束论证+回应主旨）
```

## 工作流程

### 第一步：理解研究意图

获取以下信息（从用户对话中提取，不足时提问）：
1. **研究领域**（物理学/计算机/经济学/医学/人文/其他）
2. **论文类型**（原创研究/综述/实证/理论/系统/论证）
3. **核心研究问题**（想回答什么问题）
4. **核心论点/贡献**（大致发现了什么或主张什么）

### 第二步：路由到结构原型

根据领域和论文类型选择原型：

| 领域 | 论文类型 → 原型 |
|------|----------------|
| 物理学 | 理论实验 → 原型1；综述 → 自定义 |
| 计算机 | 算法/理论 → 原型1；实证/CL/SE → 原型2；工程系统 → 原型3 |
| 经济学 | 理论模型 → 原型1；实证计量 → 原型2 |
| 医学 | 临床/流行病学 → 原型2；计算生物 → 原型2/1 |
| 人文 | 所有 → 原型4 |
| 交叉学科 | 取主导方法论对应的原型 |

### 第三步：生成引言大纲

引言必须由CARS模型的3个Move驱动：

**Move 1 — 建立领域**
- 大背景与研究问题
- 该领域的核心关注
- 问题的重要性/广泛性

**Move 2 — 建立空隙**
- 已有研究的不足
- 未被回答的关键问题
- （人文领域：用counter-claiming替代gap）

**Move 3 — 占据空隙**
- 本研究的目标
- 核心贡献/发现
- 论文结构路线图
- （计算机领域：需要numbered contribution list）
- （经济学领域：需要有roadmap paragraph）

### 第四步：生成核心章节结构

按原型模板展开核心章节结构，**在每个子章节标注其论证功能**。此阶段不评估图表——图表可行性将在后续独立步中基于数据清单判断。

#### 4.1 论证节点映射流程

对每个核心子章节（如3.1、3.2等），执行以下步骤：

**Step A — 标注论证功能**
判断该子节的核心论证功能，从以下类型中选择：
- **制度/背景阐述**：介绍制度框架、政策背景、概念定义
- **机制/因果分析**：解释A如何影响B、传导路径
- **数据/样本描述**：数据来源、样本特征、描述性统计
- **实证/结果呈现**：回归结果、估计结果、检验结果
- **对比/比较分析**：不同情境、组别、方案的对比
- **流程/设计说明**：研究设计、实验流程、方法论
- **讨论/延伸**：结果解读、政策含义、局限性

**Step B — 内容复杂度标注**
标注该子节的信息密集度（用于后续图表可行性参考，非最终决策）：
- **多维对比型**：需同时对比≥3个维度的数值信息
- **趋势/分布型**：涉及时间序列或数据分布描述
- **流程/结构型**：多步骤流程或复杂机制解释
- **单一陈述型**：单一论点或简单事实陈述

**Step C — 数据可得性预检**（仅标记，不决策）
标记该子节是否需要数据支撑：
- 如需数据，标注所需数据类型（数值对比/时间序列/结构关系）
- 最终是否画图取决于数据清单中是否有真实数据，不在本阶段决定

#### 4.2 学科特异规则（章节结构相关）

**计算机**
- Introduction末尾必须有numbered contribution list（3-5条）
- Related Work作为独立节（92%论文有此节）
- Experiments节体系化：设置→主结果→消融→分析
- 附录建议（代码/额外实验/定理证明）
- 代码/数据可用性声明

**经济学**
- Introduction末尾必须有roadmap paragraph
- 理论论文：Model→Results→Extensions
- 实证论文：Theory → Data/Research Design → Empirical Results → Policy
- 政策建议是必选项（中文A刊）
- 附录用于额外稳健性检验

**医学**
- Methods必须详尽（含power calculation）
- Results 按主要终点→次要终点组织
- Discussion独立节，不重复Results
- 讨论必须解释局限性
- CONSORT流程图（临床试验）

**人文**
- 不设单独文献综述节
- 段落标题用论题陈述句
- 结论用论证收束而非结果总结
- 章节目录用中文数字（一、二、三）或MLA格式

**物理学**
- 中文论文：引言→方法→实验结果→总结与展望
- 英文论文：Introduction→Results→Discussion→Methods（Nature风格）
- "总结与展望"几乎必含展望

### 第五步：基于数据清单的图表规划

仅在文献调研阶段产出了 `02_数据清单.md` 时执行。对第四步中标记了"需数据支撑"的子章节，检查数据清单中是否有对应的真实数据：有且维度足够可视化则规划图表（标注图表类型和数据来源文献编号），数据不足或无数据则改用文字叙述。禁止编造数据来源或使用清单外的数值。

### 第六步：校验大纲合理性

用6条底层逻辑校验，确认章节结构逻辑成立即可。

1. **确立空间** ✅ 引言是否先建立背景再进入核心贡献？
2. **分类即论证** ✅ 章节划分是否反映了对问题的分类理解？
3. **边界清晰** ✅ 文末是否有边界声明（局限性/未来工作）？
4. **问题优先** ✅ 是否以问题/新视角作为论文出发点？
5. **具体→抽象**（领域依赖）✅ 自然科学论文是否按数据→理论组织？
6. **对称嵌套**（领域依赖）✅ 自然科学论文是否有方法→实验的对称结构？

额外校验：
- 所选领域的特殊约束是否满足？
- 标题层级是否一致？
- 各节之间逻辑连贯性是否成立？

## 输入输出格式

**输入**（从用户对话中提取）：
- 研究领域
- 研究问题
- 论文类型
- 核心论点/假设
- 关键方法/数据（如有）
- 特殊要求

**输出**：
- 完整的1-2级大纲（含编号）
- 各节简要说明
- 引言的CARS锚点标注
- 领域特异规则满足情况说明
- 校验检查结果
- **图表规划清单**（每个图表的编号、类型、**数据来源文献编号**、对应数据清单条目预览——仅在有真实数据时产出，无数据不规划图表）

## 边界情况处理

- **用户只有模糊想法**：先引导确定研究问题和核心论点再生成大纲
- **交叉学科论文**：取主导方法论对应的原型，标注两个学科要求
- **综述论文**：用分类/历史逻辑展开，非IMRaD
- **中英文选择**：中文论文用中文数字编号，英文论文用阿拉伯数字
- **用户提出已有大纲**：先评估是否符合领域规范，再给出改进建议
