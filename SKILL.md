# 论文大纲生成

基于跨学科论文结构研究成果（5领域50篇真实论文章节树分析），按学科认知范式生成规范的研究论文大纲。

---

## 核心框架

### 三段式骨架（跨领域通用）

所有论文的顶层结构都是三段式：

```
引言（Introduction）→ 核心章节（Core）→ 结论（Conclusion）
```

- **引言**：建立研究空间（CARS模型驱动）
- **核心章节**：领域结构原型决定组织形式
- **结论**：收束论证/总结发现/展望未来

### 四种结构原型路由

根据用户的研究领域和论文类型，路由到对应的结构原型：

#### 原型1：论证-证明型（Claim-Evidence）
**适用领域**：物理学、经济学理论、cs.LG理论论文、数学
**核心逻辑**：提出命题→提供证明→验证结论
**典型结构**：`Introduction → Model/Theory → Experiment/Evidence → Conclusion`
**关键特征**：方法论单独成章、理论/推导单独成章、实验验证为必选项

**章节模板**：
```
1 引言
2 相关理论基础（可选：如领域已有成熟框架）
3 方法/模型
  3.1 问题形式化
  3.2 模型设计
  3.3 理论分析（定理/命题）
4 实验/验证
  4.1 实验设置
  4.2 主结果
  4.3 消融/敏感性分析
5 讨论（可选）
6 结论与展望
```

#### 原型2：流程-报告型（Process-Report）
**适用领域**：临床医学、计算生物学、部分cs.CL/cs.SE、实证社会学
**核心逻辑**：如实报告研究过程——做了什么→发现了什么→意味着什么
**典型结构**：`Introduction → Methods → Results → Discussion → Conclusion`
**关键特征**：方法论独立成章、数据说明独立、讨论节独立、最接近经典IMRaD

**章节模板**：
```
1 引言
2 文献综述（可选）
3 研究方法
  3.1 数据来源
  3.2 研究设计
  3.3 分析方法
4 研究结果
  4.1 描述性结果
  4.2 核心发现（按RQ或假设组织）
  4.3 稳健性/敏感性分析
5 讨论
6 结论
```

#### 原型3：系统-评估型（System-Evaluation）
**适用领域**：计算机工程（DB/NI/CV工程类）、软件工程、系统设计论文
**核心逻辑**：展示系统设计和实际效果
**典型结构**：`Introduction → Problem → Architecture/Design → Implementation → Evaluation → Conclusion`
**关键特征**：Problem Statement独立成章、System Architecture独立成章、Related Work位置灵活（可在末尾）

**章节模板**：
```
1 引言
2 相关工作（可在末尾）
3 问题定义/形式化
4 系统设计/方法
  4.1 整体架构
  4.2 核心模块设计
  4.3 关键算法/机制
5 实验评估
  5.1 实验设置
  5.2 主结果
  5.3 消融/效率分析
6 讨论（可选）
7 结论
```

#### 原型4：叙事-论证型（Narrative-Argument）
**适用领域**：人文学科（文学、历史、哲学、艺术史）、定性研究
**核心逻辑**：按思想发展的内在逻辑组织，段落标题=论证命题
**典型结构**：`Introduction → 论证展开(主题驱动) → Conclusion`
**关键特征**：无Methodology节、标题为论题陈述而非方法描述、章节数量灵活（3-9个）、结论名称多样

**章节模板**：
```
1 引言（问题提出+立场声明+对手立场）
2 概念辨析/语境建立
3 核心论证一（正论/概念展开）
4 核心论证二（批判/反驳已有观点）
5 回应可能的反对意见（可选）
6 结论（收束论证+回应主旨）
```

## 工作流程

### 第一步：理解研究意图

获取以下信息（从用户对话中提取，不足时提问）：
1. **研究领域**（物理学/计算机/经济学/医学/人文/其他）
2. **论文类型**（原创研究/综述/实证/理论/系统/论证）
3. **核心研究问题**（想回答什么问题）
4. **核心论点/贡献**（大致发现了什么或主张什么）

### 第二步：路由到结构原型

根据领域和论文类型选择原型：

| 领域 | 论文类型 → 原型 |
|------|----------------|
| 物理学 | 理论实验 → 原型1；综述 → 自定义 |
| 计算机 | 算法/理论 → 原型1；实证/CL/SE → 原型2；工程系统 → 原型3 |
| 经济学 | 理论模型 → 原型1；实证计量 → 原型2 |
| 医学 | 临床/流行病学 → 原型2；计算生物 → 原型2/1 |
| 人文 | 所有 → 原型4 |
| 交叉学科 | 取主导方法论对应的原型 |

### 第三步：生成引言大纲

引言必须由CARS模型的3个Move驱动：

**Move 1 — 建立领域**
- 大背景与研究问题
- 该领域的核心关注
- 问题的重要性/广泛性

**Move 2 — 建立空隙**
- 已有研究的不足
- 未被回答的关键问题
- （人文领域：用counter-claiming替代gap）

**Move 3 — 占据空隙**
- 本研究的目标
- 核心贡献/发现
- 论文结构路线图
- （计算机领域：需要numbered contribution list）
- （经济学领域：需要有roadmap paragraph）

### 第四步：生成核心章节

按原型模板展开，注意以下领域特异规则：

**计算机**
- Introduction末尾必须有numbered contribution list（3-5条）
- Related Work作为独立节（92%论文有此节）
- Experiments节体系化：设置→主结果→消融→分析
- 附录建议（代码/额外实验/定理证明）
- 代码/数据可用性声明

**经济学**
- Introduction末尾必须有roadmap paragraph
- 理论论文：Model→Results→Extensions
- 实证论文：Theory → Data/Research Design → Empirical Results → Policy
- 政策建议是必选项（中文A刊）
- 附录用于额外稳健性检验

**医学**
- Methods必须详尽（含power calculation）
- Results 按主要终点→次要终点组织
- Discussion独立节，不重复Results
- 讨论必须解释局限性
- CONSORT流程图（临床试验）

**人文**
- 不设单独文献综述节
- 段落标题用论题陈述句
- 结论用论证收束而非结果总结
- 章节目录用中文数字（一、二、三）或MLA格式
- 正文中几乎无图表

**物理学**
- 中文论文：引言→方法→实验结果→总结与展望
- 英文论文：Introduction→Results→Discussion→Methods（Nature风格）
- "总结与展望"几乎必含展望

### 第五步：数据驱动的图表规划

**核心逻辑**：文献提取真实数据 → 数据清单落盘 → 有数据才画图 → 标注真实来源 → 正文引用展开

图表不是大纲的"配图"，而是论证链中不可替代的环节。每一步决策都必须建立在已有真实数据的基础上，禁止反向编造。

**触发前提**：仅在已有真实数据时执行。无可用数据则跳过，正文全部使用文字叙述，不得使用图表占位符。

---

#### 5.1 数据盘点

对第四步产出的每个核心子章节，逐一盘查其论证所需的定量数据：

- 该子章节的论证需要什么数据？（数值对比/趋势分布/结构关系）
- 现有数据中是否有对应的真实值？
- 数据来源是什么？（具体文献/实验/官方统计，不可含混）

逐一记录：✅ 有可用数据 / ❌ 无数据 / ⚠️ 数据不完整（维度不足）

---

#### 5.2 可行性判断（四道门禁 + 降级路径）

每张图表的成立必须同时通过以下四道检查。**不通过时不直接退文字，而是按降级路径尝试更简单的呈现形式，所有路都走不通才退回文字。**

**门禁一：数据来源可溯**
- 数据必须有明确的来源文献或原始数据集
- 来源名称不可含混（不能写"据相关文献显示""某研究指出"）
- 来源中必须确实包含该数据值（不能从来源的上下文推测数值）
- ❌ 不通过 → 检查是否有其他有可靠来源的数据可替代该论证；都没有 → 改文字

**门禁二：数据值真实直接**
- 数据值必须直接来自来源，精确到原数值
- 禁止：估算值、近似值、从文字描述推测的数值
- 禁止：将"显著增长""大幅下降"等定性描述转化为定量数值
- 禁止：从图表图片中肉眼读取数值
- ❌ 不通过 → 检查是否有其他可验证数据替代；都没有 → 改文字

**门禁三：维度达标**
按以下优先级尝试，**当前形式不通则降一级，不是直接跳文字**：

| 优先级 | 呈现形式 | 最低数据要求 |
|-------|---------|------------|
| 1（首选） | 图形（折线/柱状/散点） | 满足阈值（见下） |
| 2（降级） | 表格 | ≥2个可对比数据点 |
| 3（再降） | 直接引用数据值 | ≥1个具体数值 |
| 4（兜底） | 纯文字叙述 | 无可用数据 |

**图形阈值**（优先级1的准入条件）：
- 数值对比：≥2维、≥3个可对比数据点
- 时间序列：≥3个连续时间点
- 分布/趋势：≥5个样本点

**图形不满足 → 降级为表格**（优先级2），表格维度不够 → **降级为引用具体数值**（优先级3），仅有定性描述无数据 → **纯文字**（优先级4）。

**门禁四：论证锚定**
- 每张图表必须有明确的论证功能：删掉它，论证链是否会断裂？
- 当前形式论证链不断裂 → 尝试换成对论证更关键的形式（如图形→表格让数据直接说话，或调整图表所锚定的论证节点）
- 所有形式都无法锚定到论证链 → 废弃该图表
- 同一数据不得用于多张不同图表（只保留最相关的一张）

---

#### 5.3 图表类型匹配

通过门禁后，根据数据特征选择呈现形式：

| 数据特征 | 图表形式 | 典型场景 |
|---------|---------|---------|
| 精确多维数值对比（≥3维） | 表格 | 回归结果、描述性统计、组间对比 |
| 趋势/分布/关系 | 图形（折线/柱状/散点） | 时间序列、分布特征、相关性 |
| 流程/机制/结构 | 示意图 | 传导机制、研究设计、系统架构 |
| 单一因果/简单分类 | 文字（不画图） | 理论推导、文献评述 |

---

#### 5.4 图表规划输出

每张通过门禁的图表输出以下信息：

- **图表编号**：`[图X]` / `[表X]`
- **图表类型**：统计图 / 三线表 / 示意图
- **数据来源**：具体文献编号或原始数据集名称
- **关键数据值**：直接从已有数据提取，禁止修改或近似
- **论证锚点**：对应哪个论证节点

**禁止行为（违反则整张图表无效）**：
- 数据来源不可追溯时编造文献名
- 将理论推测标注为"数据支撑"
- 维度不足时强行画图
- 使用已有数据中不存在的数值
- 为凑图表数量拆分本可合并的一张图

---

#### 5.5 正文引用约束

规划图表时同步标注其在正文中的论证位置：

- 正文应在图表附近展开对数据的解读和讨论，而非仅写"结果见图X"
- 标注图表时同时写明关键数据的含义，不假设读者自己看图
- 图表的论证结论须在正文中有对应的推理展开，不能图表在正文却未提及

### 第六步：校验大纲合理性

用6条底层逻辑校验：

1. **确立空间** ✅ 引言是否先建立背景再进入核心贡献？
2. **分类即论证** ✅ 章节划分是否反映了对问题的分类理解？
3. **边界清晰** ✅ 文末是否有边界声明（局限性/未来工作）？
4. **问题优先** ✅ 是否以问题/新视角作为论文出发点？
5. **具体→抽象**（领域依赖）✅ 自然科学论文是否按数据→理论组织？
6. **对称嵌套**（领域依赖）✅ 自然科学论文是否有方法→实验的对称结构？

额外校验：
- 所选领域的特殊约束是否满足？
- 标题层级是否一致？
- 各节之间逻辑连贯性是否成立？

## 输入输出格式

**输入**（从用户对话中提取）：
- 研究领域
- 研究问题
- 论文类型
- 核心论点/假设
- 关键方法/数据（如有）
- 特殊要求

**输出**：
- 完整的1-2级大纲（含编号）
- 各节简要说明
- 引言的CARS锚点标注
- 领域特异规则满足情况说明
- 图表规划清单（如通过门禁，含图表编号/类型/数据来源/论证锚点）
- 校验检查结果

---

## 边界情况处理

- **用户只有模糊想法**：先引导确定研究问题和核心论点再生成大纲
- **交叉学科论文**：取主导方法论对应的原型，在特异规则中标注两个学科的要求
- **综述论文**：使用分类/历史逻辑展开，而非IMRaD
- **中英文选择**：中文论文用中文数字编号（一/二/三），英文论文用阿拉伯数字（1/2/3）
- **用户提出已有大纲**：先评估是否符合领域规范，再给出改进建议
- **用户要求极简大纲**：只输出1级标题，但隐式保留2级结构在工作底稿中
