# 论文大纲生成

基于跨学科论文结构研究成果（5领域50篇真实论文章节树分析），按学科认知范式生成规范的研究论文大纲。

---

## 图表规划核心原则

**问题**：AI倾向于"每章塞一个图表"，根源是无论证目标感+无信息载体元认知。

**核心原则**：图表不是"配"的，而是论证链中不可替代的环节。本技能提供**论证需求驱动的图表决策流程**，嵌入大纲的每个子章节。

### 决策三问（在每个论证节点执行）

1. **必要性**：纯文字能否完整表达？如需>300字或需读者同时对比≥3维度，才考虑图表
2. **效率性**：图表是否比文字更高效？读者看完后能否比文字更快抓住核心信息？
3. **论证链测试**：删掉这张图/表，论证链是否会断裂？不断裂→装饰性

### 信息特征↔最优呈现形式

| 信息特征 | 最优形式 | 典型场景 |
|---------|---------|---------|
| 精确多维数值对比（≥3维） | 表格 | 回归结果、描述性统计 |
| 趋势/分布/关系 | 图形 | 时间序列、分布特征 |
| 流程/机制/结构 | 示意图 | 传导机制、研究设计 |
| 单一因果/简单分类 | 文字 | 理论推导、文献评述 |

> 经济学中表格优先于图形（精确数值>视觉冲击）；自然科学中图形更重要。

### 数据规格（每个图表节点必须输出）

每个确认需要的图表，必须同时输出数据规格字段，作为下游图表生成和正文引用的约束锚点：

| 字段 | 说明 | 示例 |
|------|------|------|
| 图表编号 | 统一编号 `[图X]`/`[表X]` | [图1] |
| 变量定义 | X/Y/分组变量，含单位与范围 | X=年份(2019-2023)，Y=营收(亿元) |
| 数据来源 | 文献引用编号/公开数据库/理论推导 | 文献[3] Table 2 |
| 数据精度 | 小数位数、百分比/绝对值 | 保留1位小数 |
| 获取方式 | 从文献引用/公开数据库/需自行计算/待调研 | 从文献引用 |
| 预期发现 | 图表预计呈现的核心趋势或对比 | A平台营收持续高于B和C |

**为什么必须输出数据规格**：大纲阶段不约束数据来源和精度，图表生成器只能自由"推测"数据，结果全是幻觉。数据规格是连接"大纲需求→图表生成→正文引用"三条链的数据锚点。

### 三段式骨架（跨领域通用）

所有论文的顶层结构都是三段式：

```
引言（Introduction）→ 核心章节（Core）→ 结论（Conclusion）
```

- **引言**：建立研究空间（CARS模型驱动）
- **核心章节**：领域结构原型决定组织形式
- **结论**：收束论证/总结发现/展望未来

### 四种结构原型路由

根据用户的研究领域和论文类型，路由到对应的结构原型：

#### 原型1：论证-证明型（Claim-Evidence）
**适用领域**：物理学、经济学理论、cs.LG理论论文、数学
**核心逻辑**：提出命题→提供证明→验证结论
**典型结构**：`Introduction → Model/Theory → Experiment/Evidence → Conclusion`
**关键特征**：方法论单独成章、理论/推导单独成章、实验验证为必选项

**章节模板**：
```
1 引言
2 相关理论基础（可选：如领域已有成熟框架）
3 方法/模型
  3.1 问题形式化
  3.2 模型设计
  3.3 理论分析（定理/命题）
4 实验/验证
  4.1 实验设置
  4.2 主结果
  4.3 消融/敏感性分析
5 讨论（可选）
6 结论与展望
```

#### 原型2：流程-报告型（Process-Report）
**适用领域**：临床医学、计算生物学、部分cs.CL/cs.SE、实证社会学
**核心逻辑**：如实报告研究过程——做了什么→发现了什么→意味着什么
**典型结构**：`Introduction → Methods → Results → Discussion → Conclusion`
**关键特征**：方法论独立成章、数据说明独立、讨论节独立、最接近经典IMRaD

**章节模板**：
```
1 引言
2 文献综述（可选）
3 研究方法
  3.1 数据来源
  3.2 研究设计
  3.3 分析方法
4 研究结果
  4.1 描述性结果
  4.2 核心发现（按RQ或假设组织）
  4.3 稳健性/敏感性分析
5 讨论
6 结论
```

#### 原型3：系统-评估型（System-Evaluation）
**适用领域**：计算机工程（DB/NI/CV工程类）、软件工程、系统设计论文
**核心逻辑**：展示系统设计和实际效果
**典型结构**：`Introduction → Problem → Architecture/Design → Implementation → Evaluation → Conclusion`
**关键特征**：Problem Statement独立成章、System Architecture独立成章、Related Work位置灵活（可在末尾）

**章节模板**：
```
1 引言
2 相关工作（可在末尾）
3 问题定义/形式化
4 系统设计/方法
  4.1 整体架构
  4.2 核心模块设计
  4.3 关键算法/机制
5 实验评估
  5.1 实验设置
  5.2 主结果
  5.3 消融/效率分析
6 讨论（可选）
7 结论
```

#### 原型4：叙事-论证型（Narrative-Argument）
**适用领域**：人文学科（文学、历史、哲学、艺术史）、定性研究
**核心逻辑**：按思想发展的内在逻辑组织，段落标题=论证命题
**典型结构**：`Introduction → 论证展开(主题驱动) → Conclusion`
**关键特征**：无Methodology节、标题为论题陈述而非方法描述、章节数量灵活（3-9个）、结论名称多样

**章节模板**：
```
1 引言（问题提出+立场声明+对手立场）
2 概念辨析/语境建立
3 核心论证一（正论/概念展开）
4 核心论证二（批判/反驳已有观点）
5 回应可能的反对意见（可选）
6 结论（收束论证+回应主旨）
```

## 工作流程

### 第一步：理解研究意图

获取以下信息（从用户对话中提取，不足时提问）：
1. **研究领域**（物理学/计算机/经济学/医学/人文/其他）
2. **论文类型**（原创研究/综述/实证/理论/系统/论证）
3. **核心研究问题**（想回答什么问题）
4. **核心论点/贡献**（大致发现了什么或主张什么）

### 第二步：路由到结构原型

根据领域和论文类型选择原型：

| 领域 | 论文类型 → 原型 |
|------|----------------|
| 物理学 | 理论实验 → 原型1；综述 → 自定义 |
| 计算机 | 算法/理论 → 原型1；实证/CL/SE → 原型2；工程系统 → 原型3 |
| 经济学 | 理论模型 → 原型1；实证计量 → 原型2 |
| 医学 | 临床/流行病学 → 原型2；计算生物 → 原型2/1 |
| 人文 | 所有 → 原型4 |
| 交叉学科 | 取主导方法论对应的原型 |

### 第三步：生成引言大纲

引言必须由CARS模型的3个Move驱动：

**Move 1 — 建立领域**
- 大背景与研究问题
- 该领域的核心关注
- 问题的重要性/广泛性

**Move 2 — 建立空隙**
- 已有研究的不足
- 未被回答的关键问题
- （人文领域：用counter-claiming替代gap）

**Move 3 — 占据空隙**
- 本研究的目标
- 核心贡献/发现
- 论文结构路线图
- （计算机领域：需要numbered contribution list）
- （经济学领域：需要有roadmap paragraph）

### 第四步：生成核心章节（含论证节点→图表需求映射）

按原型模板展开核心章节结构，同时**在每个子章节标注其论证功能，并评估是否需要图表支持**。这不是给每个章节硬塞一张图，而是从论证需求逆推信息呈现形式。

#### 4.1 论证节点映射流程

对每个核心子章节（如3.1、3.2等），执行以下两步：

**Step A — 标注论证功能**
判断该子节的核心论证功能，从以下类型中选择：
- **制度/背景阐述**：介绍制度框架、政策背景、概念定义
- **机制/因果分析**：解释A如何影响B、传导路径
- **数据/样本描述**：数据来源、样本特征、描述性统计
- **实证/结果呈现**：回归结果、估计结果、检验结果
- **对比/比较分析**：不同情境、组别、方案的对比
- **流程/设计说明**：研究设计、实验流程、方法论
- **讨论/延伸**：结果解读、政策含义、局限性

**Step B — 信息承载形式评估**

| 论证功能 | 典型内容 | 图表需求判断 |
|---------|---------|------------|
| 制度/背景阐述 | 政策框架、制度演变 | ≥3个制度维度或跨期→示意图 |
| 机制/因果分析 | 传导路径、影响机制 | ≥2步传导链→机制图 |
| 数据/样本描述 | 变量定义、统计特征 | 描述性统计表为经济学**必选** |
| 实证/结果呈现 | 回归表、估计结果 | 回归结果表为经济学**必选**，辅助用系数图 |
| 对比/比较分析 | 组间对比、稳健性检验 | ≥3组→表格，≤3→文字 |
| 流程/设计说明 | 研究设计、数据处理 | 多步骤→流程图 |
| 讨论/延伸 | 结果解读、政策含义 | 一般无需图表 |

**Step C — 必要性三问自检**
对Step B中标记为"潜在图表"需求的每项，逐项执行决策三问（见"图表规划核心原则"一节）：
- 纯文字能否说清楚？（不通过→需要图表）
- 图表比文字更高效？（不通过→只需文字提一下数据）
- 删掉后论证链是否完整？（断裂→必需；完整→装饰性，建议删除）

仅在Step C三项都通过时才确认需要图表。

**Step D — 图表数据规格设计**
对Step C确认需要的每项图表，输出数据规格：
1. **变量定义**：明确X轴、Y轴、分组变量的名称、单位和取值范围。例："X轴=年份(2019-2023)，Y轴=营收(亿元)，分组=平台A/B/C"
2. **数据来源**：引用文献编号、公开数据库名称，或标注"理论推导值/模拟数据（需确认）"。禁止不标注数据来源
3. **数据精度**：小数位数（如保留1位小数）、数据格式（如百分比/绝对值）
4. **获取方式**：从文献引用/从公开数据库/需自行计算/待调研
5. **预期发现**：该图表预计呈现的核心趋势或对比关系，供正文作者明确引用落脚点

#### 4.2 领域特异规则中的图表要求

在原领域特异规则基础上，增加图表相关约束：

**计算机**
- 实验设置建议用表呈现（超参数、数据集统计）
- 主结果建议用图呈现（柱状图/折线图）
- 消融分析建议用图呈现
- 架构图在System Design节为必选项

**经济学**
- 描述性统计表（Table 1）为必选项
- 主回归结果表为必选项
- 稳健性检验结果表为必选项
- 机制分析路径图（可选但推荐）
- 时间序列折线图（如政策实施前后的变化趋势）
- 图形应简洁（黑白/灰度优先，避免3D效果和过度配色）
- **核心原则**：经济学论文中表格优先于图形，精确数值比视觉冲击更重要

**医学**
- 受试者流程图（CONSORT）为必选项
- 基线特征表（Table 1）为必选项
- 主要和次要终点结果表为必选项
- Kaplan-Meier曲线（生存分析）
- 森林图（荟萃分析）

**人文**
- 正文中几乎无图表（极少数情况：历史地图、文本分析示意图）
- 图表不列入大纲规划，仅在写作阶段如有需要再添加
- 不强制要求任何图表

**物理学**
- 实验装置示意图推荐
- 结果图（光谱/曲线/图像）为核心呈现形式
- 数据表作为补充

#### 4.3 学科特异规则（仅含非图表部分，图表要求见4.2）

**计算机**
- Introduction末尾必须有numbered contribution list（3-5条）
- Related Work作为独立节（92%论文有此节）
- Experiments节体系化：设置→主结果→消融→分析
- 附录建议（代码/额外实验/定理证明）
- 代码/数据可用性声明

**经济学**
- Introduction末尾必须有roadmap paragraph
- 理论论文：Model→Results→Extensions
- 实证论文：Theory → Data/Research Design → Empirical Results → Policy
- 政策建议是必选项（中文A刊）
- 附录用于额外稳健性检验

**医学**
- Methods必须详尽（含power calculation）
- Results 按主要终点→次要终点组织
- Discussion独立节，不重复Results
- 讨论必须解释局限性
- CONSORT流程图（临床试验）

**人文**
- 不设单独文献综述节
- 段落标题用论题陈述句
- 结论用论证收束而非结果总结
- 章节目录用中文数字（一、二、三）或MLA格式

**物理学**
- 中文论文：引言→方法→实验结果→总结与展望
- 英文论文：Introduction→Results→Discussion→Methods（Nature风格）
- "总结与展望"几乎必含展望

### 第五步：校验大纲合理性（含图表规划校验）

用6条底层逻辑校验：

1. **确立空间** ✅ 引言是否先建立背景再进入核心贡献？
2. **分类即论证** ✅ 章节划分是否反映了对问题的分类理解？
3. **边界清晰** ✅ 文末是否有边界声明（局限性/未来工作）？
4. **问题优先** ✅ 是否以问题/新视角作为论文出发点？
5. **具体→抽象**（领域依赖）✅ 自然科学论文是否按数据→理论组织？
6. **对称嵌套**（领域依赖）✅ 自然科学论文是否有方法→实验的对称结构？

**图表规划专项校验**：

7. **论证必要性校验** ✅ 每个拟议图表是否满足决策三问（必要→高效→链断裂）？不满足三问中的任何一问，重新评估
8. **领域合规与形式选择** ✅ 是否符合该领域图表规范？形式（表格/图形/示意图）是否符合信息特征（精确对比→表；趋势分布→图；流程结构→示意图）？

额外校验：
- 所选领域的特殊约束是否满足？
- 标题层级是否一致？
- 各节之间逻辑连贯性是否成立？

## 输入输出格式

**输入**（从用户对话中提取）：
- 研究领域
- 研究问题
- 论文类型
- 核心论点/假设
- 关键方法/数据（如有）
- 特殊要求

**输出**：
- 完整的1-2级大纲（含编号）
- 各节简要说明
- 引言的CARS锚点标注
- 领域特异规则满足情况说明
- 校验检查结果
- **图表规划清单**（每个拟议图表的用途、呈现形式、论证必要性说明、满足决策三问的哪几问、**数据规格：变量定义/数据来源/数据精度/获取方式/预期发现**）

## 边界情况处理

- **用户只有模糊想法**：先引导确定研究问题和核心论点再生成大纲
- **交叉学科论文**：取主导方法论对应的原型，标注两个学科要求
- **综述论文**：用分类/历史逻辑展开，非IMRaD
- **中英文选择**：中文论文用中文数字编号，英文论文用阿拉伯数字
- **用户提出已有大纲**：先评估是否符合领域规范，再给出改进建议
